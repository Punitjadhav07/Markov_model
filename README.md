# Customer Purchase Behavior Analysis (Markov Chain)

[![Python 3.12+](https://img.shields.io/badge/python-3.12+-blue.svg)](https://www.python.org/downloads/)
[![Streamlit](https://img.shields.io/badge/app-Streamlit-FF4B4B.svg)](https://streamlit.io/)

End-to-end analysis of **Retailrocket-style** event logs (`view`, `addtocart`, `transaction`). The project builds a **first-order Markov transition matrix** from per-user journeys, validates row stochasticity, exports artifacts for a **Streamlit** predictor, and documents **funnel drop-off** metrics.

**Repository:** [github.com/Punitjadhav07/Markov_model](https://github.com/Punitjadhav07/Markov_model)

---

## What this project does

| Step | Where | Output |
|------|--------|--------|
| Clean & order events | `notebooks/analysis.ipynb` | Filtered events, datetime index |
| Map events → states | Notebook | `Browse`, `Add_to_Cart`, `Purchase` |
| Build sequences + **Exit** | Notebook | One sequence per `visitorid`, **Exit** appended (absorbing end) |
| Count transitions | Notebook | Pair counts → crosstab |
| Normalize rows | Notebook | `P(next \| current)`; **Exit row dropped** (no outgoing transitions) |
| Validate | Notebook | Row sums ≈ 1.0 |
| Visualize & save | Notebook | `app/heatmap.png`, `transition_probs.pkl`, `transition_probs.csv` |
| Serve predictions | `app/app.py` | Loads **saved** matrix only (no recomputation) |

## Markov model (how it is used)

1. **States** — Raw events map to behavior states:
   - `view` → `Browse`
   - `addtocart` → `Add_to_Cart`
   - `transaction` → `Purchase`

2. **Terminal state** — Each user sequence ends with **`Exit`** (session end). Transitions like `Purchase → Exit` are counted; **`Exit` is not kept as a row** in the probability matrix because it is absorbing (no “next state” distribution to normalize).

3. **Transitions** — For each sequence, consecutive pairs `(from_state, to_state)` are collected with `zip(seq[:-1], seq[1:])`.

4. **Counts** — `pd.crosstab(from_state, to_state)` builds the count matrix. The **Exit** row is dropped before normalization.

5. **Probabilities** — Each remaining row is divided by its sum so **each row sums to 1** (valid Markov transition matrix for non-terminal states).

6. **Use** — The matrix supports:
   - **Next-state prediction** — `predict_next_state(current_state)` returns a sorted probability series.
   - **Funnel interpretation** — e.g. `Browse → Exit` (drop-off), `Browse → Add_to_Cart`, `Add_to_Cart → Purchase`, `Add_to_Cart → Exit` (cart abandonment).

## Notebook (`notebooks/analysis.ipynb`)

Run **all cells top to bottom** (kernel: project `.venv`). The notebook:

- Loads `../Data/events.csv`, filters events, converts millisecond timestamps to datetime, sorts by `visitorid`, `timestamp`.
- Maps to `state`, builds sequences with **`Exit`** appended.
- Builds transition counts, drops **`Exit` from the index**, normalizes to **`transition_probs`**.
- **Asserts** row sums equal 1 (within `1e-6`).
- Plots a **seaborn** heatmap and saves **`../app/heatmap.png`**.
- Defines **`predict_next_state`** and prints sample predictions.
- Computes **business metrics** (unique visitors, purchasers, conversion rates, funnel lines from the matrix).
- Saves **`../app/transition_probs.pkl`** and **`../app/transition_probs.csv`**.

> **Data:** Place `events.csv` under `Data/` locally. It is listed in `.gitignore` so the raw file is not pushed to GitHub.

## Streamlit app (`app/app.py`)

- **Page title:** Customer Behavior Predictor (`layout="centered"`).
- **Data source:** Loads **`transition_probs.pkl`** from the same directory as `app.py` (cached). If missing, falls back to **`transition_probs.csv`**. If neither exists, shows an error: run the notebook first.
- **Does not** rebuild the Markov matrix in the app — probabilities always come from saved artifacts.
- **Sections:** Next-state prediction (select state, primary button, matplotlib horizontal bar chart with highlighted top outcome + `st.dataframe` + interpretation `st.info`), full seaborn heatmap, raw matrix in an expander (styled table), funnel **metrics** (Browse→Exit, Browse→Add_to_Cart, Add_to_Cart→Purchase) from the loaded matrix.

## Setup

```bash
cd customer_prediction
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Place **`Data/events.csv`** (Retailrocket events) in `Data/`, then run the notebook to regenerate `app/transition_probs.*` and `app/heatmap.png` if needed.

---

## Run

**1. Notebook (required first for a fresh clone without artifacts)**

```bash
jupyter notebook notebooks/analysis.ipynb
```

**2. Streamlit**

```bash
streamlit run app/app.py
```

Open the local URL shown in the terminal (typically `http://localhost:8501`).
